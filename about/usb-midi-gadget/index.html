Configure Raspberry Pi as a USB MIDI device for direct connection to MIDI hardware

USB MIDI Gadget Setup

This guide shows you how to configure your Raspberry Pi 4 to appear as a USB MIDI device, allowing MIDI hardware like the Akai MPC One+ to recognize it directly over USB without requiring a separate MIDI adapter.

This is an optional advanced setup that enables:

  • Direct USB connection between Pi and MIDI hardware
  • Power and communication over a single USB-C cable
  • Elimination of external MIDI adapters

Prerequisites

Make sure your Pi is up to date and has the necessary tools installed:

sudo apt update && sudo apt upgrade -y
sudo apt install alsa-utils

Step 1: Enable the DWC2 Driver

The Pi 4's USB-C port needs to be switched into peripheral (device) mode so it can present itself as a MIDI device to your MIDI hardware.

Edit the boot config:

sudo nano /boot/firmware/config.txt

Add the following line at the bottom:

dtoverlay=dwc2,dr_mode=peripheral

Then edit the kernel modules file:

sudo nano /etc/modules

Add these two lines:

dwc2
g_midi

Reboot for the changes to take effect:

sudo reboot

Step 2: Verify the Module Loaded

After rebooting, confirm the g_midi module loaded successfully:

lsmod | grep g_midi

Then check that ALSA has registered a new MIDI port:

aconnect -l

You should see output similar to:

client 20: 'MIDI Gadget' [type=kernel]
    0 'MIDI Gadget MIDI 1'

If the gadget port does not appear, check the kernel log for errors:

dmesg | grep dwc2
dmesg | grep g_midi

Also verify the gadget appears as a sound card:

cat /proc/asound/cards

Note: Make sure you are using a USB-C cable that supports data, not a charge-only cable. Charge-only cables will not work.

Step 3: Test the Gadget Port

Before connecting to your MIDI hardware, verify the gadget port is functioning by listing available MIDI hardware ports:

amidi -l

You should see the gadget listed, with a hardware address like hw:2,0,0:

Dir Device    Name
IO  hw:2,0,0  f_midi

The device name f_midi confirms the USB MIDI gadget is active and ready. This is the port name Sketchatone will use in the next step.

Step 4: Configure Sketchatone

Update Sketchatone's config.json to route MIDI output through the USB gadget port.

Locate and open your config.json file, then update the midi section as follows:

{
  "midi": {
    "midi_output_backend": "rtmidi",
    "midi_output_id": "f_midi",
    "midi_input_id": 1,
    "midi_input_exclude": ["sketchatone", "ZynMidiRouter", "zynseq", "zynsmf", "ttymidi", "Midi Through", "RtMidi"],
    "jack_client_name": "sketchatone",
    "jack_auto_connect": "chain0",
    "note_duration": 1.5,
    "midiInterMessageDelay": 0
  }
}

The key change is midi_output_id, which is updated from 1 to "f_midi" — the name of the USB MIDI gadget port identified in Step 3.

Save the file and restart Sketchatone for the change to take effect.

Step 5: Connect to Your MIDI Hardware

Plug a USB-C cable from the Pi into your MIDI hardware's USB port (e.g., Akai MPC One+).

On your MIDI hardware, navigate to the MIDI settings (for MPC One+: Settings → MIDI → USB).

The Pi should appear as a MIDI device. Once recognized, MIDI events generated by Sketchatone from your drawing tablet will be sent directly to your MIDI hardware over USB.

Troubleshooting

Issue What to check
g_midi not in lsmod Verify /etc/modules has both dwc2 and g_midi, then reboot
f_midi not in amidi -l Check dmesg for DWC2 or g_midi errors
MIDI hardware doesn't recognize the Pi Try a different USB-C cable (must be data-capable)
Sketchatone can't find f_midi Confirm the gadget port is visible via amidi -l before starting Sketchatone

Additional Notes

Power Considerations

When using USB gadget mode, the Pi can be powered through the USB-C connection from some MIDI hardware devices. However, this depends on:

  • Whether your MIDI hardware provides sufficient power over USB (typically 5V/2.5A minimum for Pi 4)
  • The quality of your USB-C cable

If the Pi doesn't power on or shows undervoltage warnings, you may need to use a separate power supply.

Reverting to Host Mode

If you need to switch the Pi back to normal USB host mode (to use USB peripherals), remove the dtoverlay=dwc2,dr_mode=peripheral line from /boot/firmware/config.txt and reboot.
