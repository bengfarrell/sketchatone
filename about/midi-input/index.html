Using external MIDI keyboards to set chord notes in Sketchatone

MIDI Input

Sketchatone can receive MIDI input from external keyboards or controllers to dynamically set the notes you strum. This allows you to play chords on a keyboard while strumming them on your tablet.

How It Works

When MIDI input is enabled:

  • ✅ Hold notes on your MIDI keyboard to set the strum notes
  • ✅ Strum on your tablet to play those notes
  • ✅ Change chords by pressing different keys
  • ✅ The last held chord persists when you release all keys

Automatic Connection

Sketchatone automatically connects to all available MIDI input sources, except those that could cause feedback loops. This "connect all" approach means you don't need to manually select which MIDI device to use - just plug in your keyboard and start playing.

The web dashboard shows the connection status and displays which device sent the most recent MIDI notes.

Feedback Loop Prevention

To prevent MIDI feedback loops (where Sketchatone's output gets routed back to its input), certain ports are automatically excluded from auto-connection. This is controlled by the midi_input_exclude configuration option.

Default Exclusion List

The following port name patterns are excluded by default:

PatternReason
sketchatoneOur own output port
Midi ThroughALSA Midi Through (loopback)

Zynthian-Specific Exclusions

On Zynthian, additional patterns should be excluded to prevent feedback through the internal MIDI routing:

{
  "midi": {
    "midi_input_exclude": [
      "sketchatone",
      "Midi Through",
      "ZynMidiRouter",
      "zynseq",
      "zynsmf",
      "ttymidi"
    ]
  }
}

How Exclusion Works

Port names are matched case-insensitively as substrings. For example, the pattern "sketchatone" will exclude any port containing "sketchatone" in its name, such as sketchatone:midi_out or Sketchatone Output.

Backend Differences

Python/JACK (Zynthian)

On Zynthian and other JACK-based systems, Sketchatone uses JACK MIDI for both input and output. It automatically connects to all JACK MIDI output ports (which are input sources for Sketchatone), except those matching the exclusion patterns.

Node.js/RtMidi (macOS, Windows, Linux)

On other platforms, Sketchatone uses RtMidi which connects directly to all available MIDI input ports, except those matching the exclusion patterns.

Virtual Port Mode

Both backends support a "register only" mode where Sketchatone creates a virtual MIDI input port without auto-connecting to any sources. This is useful when:

  • You want to configure MIDI routing externally (e.g., via Zynthian's MIDI routing UI)
  • You have a complex MIDI setup that requires manual routing

Troubleshooting

No MIDI Input Detected

  1. Check that your MIDI device is connected and powered on
  2. Verify the device appears in your system's MIDI settings
  3. Check the server logs for "Connecting to all ports" messages

Feedback Loop (Notes Playing Continuously)

  1. Add the problematic port pattern to midi_input_exclude in your config
  2. On Zynthian, make sure ZynMidiRouter is in the exclusion list
  3. Restart the service after changing the config

Specific Device Not Working

  1. Check if the device name matches any exclusion pattern
  2. Remove or modify the exclusion pattern if needed
  3. Check server logs to see which ports are being connected/excluded
