Understanding Sketchatone's configuration files

Configuration Overview

Sketchatone uses a single combined configuration file that includes strummer settings, MIDI options, and server configuration. The device (tablet) configuration is referenced via the server.device field.

Combined Configuration File

The main config file contains all strummer, MIDI, and server settings. Use the -c or --config flag to specify it.

Example: sample-config.json

{
  "strumming": {
    "chord": "Am",
    "midi_channel": 0
  },
  "note_velocity": {
    "control": "pressure",
    "min": 0,
    "max": 127
  },
  "midi": {
    "midi_output_backend": "rtmidi",
    "midi_output_id": 2
  },
  "server": {
    "device": "devices",
    "http_port": 3000,
    "ws_port": 8081
  }
}

Device Configuration

The server.device field specifies where to find tablet configuration files. It supports:

  • Relative paths - Resolved from the config file's directory (e.g., "devices")
  • Absolute paths - Used as-is (e.g., "/opt/sketchatone/configs/devices")
  • Direct file paths - Skip auto-detection (e.g., "/path/to/xp-pen.json")

Device config files define how to read HID data from your graphics tablet and are matched automatically based on USB vendor/product IDs.

Example: devices/xp-pen-deco-640.json

{
  "name": "XP Pen Deco 640",
  "vendorId": "0x28bd",
  "productId": "0x2904",
  "capabilities": {
    "hasPressure": true,
    "pressureLevels": 16384,
    "hasTilt": true
  },
  "byteCodeMappings": { ... }
}

Configuration Sections

Section Purpose
strumming Core strummer settings (chord, notes, channel)
note_velocity Velocity parameter mapping
note_duration Duration parameter mapping
pitch_bend Pitch bend parameter mapping
note_repeater Tremolo/repeat settings
transpose Transpose settings
stylus_buttons Pen button actions
strum_release Release trigger settings
tablet_buttons Chord progression mapping
midi MIDI backend settings
server Server settings
